# Gradient Stage + Wizard Decomposition Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Gradient Stage backdrop and a state-driven Checklist to the design system — as separate, composable static-CSS + Vue components — then document the full onboarding wizard as a composition of them with existing primitives.

**Architecture:** Two new primitives. The **Checklist** (`.ds-checklist` + `DsChecklist`) is a normal component living in `components.css`. The **Gradient Stage** (`.ptn-stage` + `DsStage`) is a screen-level recipe living in `patterns.css`, with a thin Vue wrapper. The original wizard's Tauri/Bluetooth logic is NOT ported — these are presentational; state is owned by the consumer. The wizard is shown only as a docs composition (`.ptn-stage` + `.ds-panel` + `.ds-checklist` + `.ds-btn`).

**Tech Stack:** Plain CSS (tokens-driven), Vue 3 `<script setup lang="ts">`, Vitest + @vue/test-utils, Storybook (CSF3), static multi-page docs assembled from `docs/nav.mjs`.

## Global Constraints

- **Token vocabulary is THIS repo's, not the source app's.** Use: `--bg` (body bg), `--accent-soft` (glow), `--success`, `--danger`, `--text` (primary text), `--text-3` (muted), `--border-soft` (subtle border), `--radius` (8px). Do NOT use `--bg-body`, `--text-dim`, `--status-success`, `--status-danger`, `--border-subtle` — they do not exist.
- **No new icons.** Only `check`, `close`, `warning` (all already in `icons/approved.json`). The `running` state uses the existing spinner (`.ds-spinner` / `DsSpinner`), not an icon. `pending` uses a CSS-drawn hollow dot, no icon.
- **No new design tokens** — everything composes from existing `css/tokens.css`.
- Reduced motion is already handled globally (`css/base.css:45` suppresses `ds-spin`). Do not add per-component reduced-motion CSS.
- Class grammar: primitives `.ds-*`, recipes `.ptn-*`, states `data-state`/`.is-*`. Two-space HTML indentation. Compact CSS rules matching `components.css` style.
- Vue components register in `vue/index.ts`; shared types live in `vue/types.ts`; each component ships a `.test.ts` and a `.stories.ts`.
- `cssHas()` (`vue/__support__/css.ts`) reads ONLY `css/components.css`. It can assert `.ds-checklist*` classes but CANNOT assert `.ptn-stage` (that lives in `patterns.css`). The Stage test must assert rendered DOM instead.
- The coverage gate (`tools/coverage-core.mjs`, run by `npm run build` → `coverage:check`) flags any primitive with zero examples in `pages/*.html`. Every new primitive needs a standalone example.

import type { ChecklistItem, ChecklistState } from "../types";
import DsIcon from "./DsIcon.vue";
import DsSpinner from "./DsSpinner.vue";

defineProps<{ items: ChecklistItem[] }>();

const ICON: Partial<Record<ChecklistState, string>> = {

  warn: "warning",
  error: "close",

<template>
  <ul class="ds-checklist" aria-live="polite">
    <li
      v-for="item in items"
      :key="item.id"
      class="ds-checklist-item"
      :data-state="item.state"
    >
      <span class="ds-checklist-ico">
        <DsSpinner v-if="item.state === 'running'" />
        <DsIcon v-else-if="ICON[item.state]" :name="ICON[item.state]!" :size="22" />
      </span>
      <span class="ds-checklist-text">
        <span class="ds-checklist-title">{{ item.title }}</span>
        <span v-if="item.note" class="ds-checklist-note">{{ item.note }}</span>
      </span>
    </li>
