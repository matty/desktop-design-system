Desktop Design System

A desktop-native design language and CSS grammar. Near-black surfaces, white & grey ink, and primary controls that follow the OS system accent by default — on Windows, that means the selected Windows accent color. Compose tokens, primitives, utilities, states, and full-screen patterns; switch to light, set a brand accent override, or reserve semantic colors for explicit meaning. Danger red is reserved for destructive actions and window-close.

Principles

System accent by default

Primary controls use AccentColor and AccentColorText, so the UI inherits the platform accent while state still relies on brightness and weight.

Desktop density

Compact 34px controls, hairline dividers, top-left aligned content. It reads like a utility, not a website.

Mono for data

Numbers, IDs, addresses and live values use JetBrains Mono so they stay legible and aligned. Sora carries everything else.

The shell

An app shell composed from design-system primitives, utilities, and states.
