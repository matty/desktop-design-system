Keyboard Interaction Contracts

Every interactive component follows a consistent keyboard model. This page documents the expected key bindings per component so implementations and tests can verify conformance.

Interaction contracts

ComponentKeysBehavior
Menu / Context menu ↑ ↓ Enter Esc → ← Roving focus; Enter activates; Esc closes and returns focus; → opens submenu, ← closes it.
Combobox Space Enter Esc type Space / Enter opens; type to filter (filterable); Esc closes; click an option to select.
Tree ↑ ↓ → ← Home End Up/Down move; → expands, ← collapses or moves to parent; Home/End jump; type-ahead.
Dialog Tab Esc Focus trapped within; Esc closes and returns focus to the opener.
Tabs / Segmented ← → Roving focus across the group.
Resizable splitter ← → / ↑ ↓ When focused, arrows resize; double-click resets.
Command palette Ctrl+K Esc Ctrl/⌘+K opens; type to filter; Esc closes.
All roving-tabindex groups expose a single tabindex="0" stop; arrow keys move the active item and wrap. dsFocusTrap(container) constrains Tab within a dialog; dsRovingTabindex(container, opts) wires arrow navigation for menus, trees, and tabs.

JS utilities

Two primitives — exposed as window.dsFocusTrap and window.dsRovingTabindex — cover the two dominant keyboard patterns. All higher-level components (context menu, combo, tree, splitter) consume them.

dsFocusTrap(container)

Constrains Tab/Shift-Tab within container. Returns { release() } — call it to remove the trap and restore focus to the previously focused element.

dsRovingTabindex(container, opts)

Wires Up/Down (or Left/Right) arrow navigation across matching children. opts.selector defaults to [role="menuitem"],[role="treeitem"],[role="option"]. opts.orientation is "vertical", "horizontal", or "both". opts.onActivate(el) fires on Enter/Space. Returns { destroy(), focusFirst() }.

Both utilities are consumed by Tasks B1 (context menu), B2 (combo), B3 (tree), and C1 (splitter). They are intentionally low-level — they wire keyboard events only, leaving ARIA attributes to the component markup.
